## 1) Question

Given an array of integers `nums` and an integer `target`, return *indices of the two numbers such that they add up to `target`*.

You may assume that each input would have **exactly one solution**, and you may not use the *same* element twice.

You can return the answer in any order.

**Example 1:**

```structured text
Input: nums = [2,7,11,15], target = 9
Output: [0,1]
Explanation: Because nums[0] + nums[1] == 9, we return [0,1].
```

**Example 2:**

```structured text
Input: nums = [3,2,4], target = 6
Output: [1,2]
```

**Example 3:**

```structured text
Input: nums = [3,3], target = 6
Output: [0,1]
```

**Constraints:**

- `2 <= nums.length <= 104`

- `-109 <= nums[i] <= 109`

- `-109 <= target <= 109`

- **Only one valid answer exists.**

## 2) Solution

### (1) One-Pass Hash Table

- Time Complexity:  $$ O(n) $$
- Space Complexity: $$ O(n) $$

#### a. C++

- Runtime: 0 ms
- Memory: 14.76 MB

```c++
class Solution {
 public:
  vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int, int> numToIndex;

    for (int i = 0; i < nums.size(); ++i) {
      if (const auto it = numToIndex.find(target - nums[i]); // 'find': iterator
          // If Iterator 'find' doesn't reach the past of the element (better than end())
          it != numToIndex.cend()) 
        return {it->second, i}; // it->first: key, it->second: value
      numToIndex[nums[i]] = i;
    }

    throw;
  }
};

```

#### b. Go

- Runtime: 0 ms

```Go
func twoSum(nums []int, target int) []int {
    // preallocate map capacity
    numToIdx := make(map[int]int, len(nums)) 
    for i, num := range nums{
        // ok: map lookups, 
        // 'true' if the key exists in the map,
        if index, ok := numToIdx[target - num]; ok{ 
            return []int{index, i}
        	}
        numToIdx[num] = i
    	}
    return nil
	}
}
```

### (2) Two-Pass Hash Table

- Time Complexity: $$ O(n) $$
- Space Complexity: $$ O(n) $$

- Runtime: 6 ms
- Memory: 15.91 MB

```C++
class Solution {
public:
    vector<int> twoSum(vector<int>& nums,   int target) {
    unordered_map<int, int> numMap;
    for (int i = 0; i < nums.size(); i++) {
        numMap[nums[i]] = i;
    }
    for (int i = 0; i < nums.size(); i++) {
        int complement = target - nums[i];
        if (numMap.count(complement) && numMap[complement] != i) {
            return {i, numMap[complement]};
        }
    }
    return {};
  }
};
```

### (3) Brute Force

- Time Complexity: $$ O(n^2) $$
- Space Complexity: $$ O(1) $$

- Runtime: 41ms
- Memory: 13.94 MB

```c++
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
    for (int i = 0; i < nums.size(); i++) {
        for (int j = i + 1; j < nums.size(); j++) {
            if (nums[i] + nums[j] == target) {
                return {i, j};
            }
        }
    }
    return {};
  }
};
```

## 3) My code

- Time Complexity: $$ O(n^2) $$
  - Tries all pairs `(i, j)`
- Space Complexity: $$ O(n^2) $$
  - Recursive calls without tail-call optimization
- Runtime: 148 ms
- Memory: 14.15 MB


```c++
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        vector<int> result;
        int i = 0;
        int j = 1;
        backtrack(nums, target, i, j);
        result.push_back(i);
        result.push_back(j);
        return result;
    };

    void backtrack(vector<int>& nums, int target, int& x, int& y){
        vector<int> result;
        int sum = 0;
        sum = nums[x] + nums[y];
        if(sum == target){
            return;
        };

        y++;
        if(y+1 > nums.size()){
            x++;
            y=x+1;
        };
        backtrack(nums, target, x, y);
    };
};
```
